Say It with Charts

Gene Zelazny / McKinsey & Company  ·  五步法完整手册
核心原则

先确定你想说什么，再决定怎么说。
图表是观点的载体，不是数据的仓库。

1

明确你的信息

Message First

在打开 Excel 之前，先回答这一个问题：

自问

「我希望观众看完这张图之后，记住什么？」

常见错误：大多数人先画图再想说什么。结果是图表展示了数据，却没有传递观点。Zelazny 的起点正好相反——先写一句话观点，再选图表形式。

3

选择具体图表形式

Chart Form

根据比较类型 + 数据特征，快速定位最合适的图表。

时间序列
数据点多还是少？
数据点多 / 强调趋势 → 折线图
数据点少 / 强调量级 → 柱状图
多系列叠加 → 堆积面积图
排名对比
标签长短？数量多少？
标签短、项目≤6 → 纵向柱状图
标签长 / 项目多 → 横向条形图
正负值对比 → 瀑布图
构成比例
切片数量？是否有时间维度？
≤6个切片、静态 → 饼图
>6个切片 → 改用条形图
多时间点构成变化 → 堆积柱状图
相关性
两个变量的关系
两变量相关 → 散点图
加第三维度 → 气泡图
4

写行动标题

Action Title

标题不是图表内容的描述，而是你的结论。这是 Zelazny 最核心的一条规则。

✗ 描述性标题（错误）
各地区季度销售额
只说「是什么」，读者还要自己分析得出结论。
✓ 行动性标题（正确）
华北区连续三季度领跑，西南区需重点关注
直接给出「所以什么」，图表作为证据支撑这句话。
✗
用户增长趋势（2022–2024）
数据范围描述，没有洞察。
✓
付费用户增速在 Q3 放缓，须在年底前找到新的增长点
包含时间、现象、以及隐含的行动建议。
写标题的方法：先写「数据显示……」开头的一句话，然后删掉「数据显示」，剩下的就是你的行动标题。
5

简化设计

Simplify

每一个视觉元素都应该服务于你的信息。无法传递信息的元素，一律删除。

✗
删除网格线 轻量参考线代替，或直接标注数值
✗
删除 3D 效果 3D 柱/饼让数据失真，读者难以准确比较
✗
删除装饰性颜色 颜色只用于区分类别或强调关键数据
✗
删除图例（如可能） 直接在数据系列旁标注名称，减少视线跳动
✓
保留关键数据标签 最高/最低值、转折点直接标注在图上
✓
保留数据来源 在图表左下角注明数据来源，建立可信度
✓
保留 Y 轴零基线 柱状图必须从零开始，否则视觉上夸大差异
✓
保留一致的字体层级 标题 > 轴标签 > 数据标注，层级清晰
⇄

改前 vs 改后：真实案例

Before / After

同一份数据，用错误方式和正确方式呈现，传递效果截然不同。

案例 1：构成比例——切片过多的饼图 → 条形图
✗ 改前：8片饼图，无法比较
各渠道流量占比
切片过多、颜色杂乱，读者无法快速判断哪个渠道更重要。
✓ 改后：横向条形图，按大小排序
自然搜索是最大流量来源，占比近三成
按值降序排列，关键渠道一目了然，无需图例。
案例 2：时间序列——杂乱折线图 → 聚焦关键信息
✗ 改前：五条线挤在一起，无重点
各产品线月度收入（万元）
五条颜色相近的线、密集网格线、无标注——读者不知道该看哪里。
✓ 改后：突出主角，灰化其他
产品C自Q3起逆势增长，已超越其他产品线
主线加粗加色，其余灰化；关键转折点直接标注数值。
案例 3：坐标轴——Y轴截断夸大差异 → 从零开始
✗ 改前：Y轴从80开始，差异被放大
四城市用户满意度评分
视觉上北京"高出一倍"，实际差距仅4分。这是数据操纵的常见手法。
✓ 改后：从零起始，差距还原真实比例
四城满意度接近，北京略高（88 vs 84）
柱状图必须从零开始。差距真实但细微——结论也要跟着改。
案例 4：图表类型——用折线图比较无序类别 → 条形图
✗ 改前：折线图暗示了不存在的趋势
各部门年度预算（万元）
折线图用于连续时间数据。部门之间没有顺序关系，折线制造了虚假趋势。
✓ 改后：条形图，按预算大小排序
研发部预算最高，行政部仅为其四分之一
条形图排序后，差距清晰，最大/最小值对比一眼可见。

常见错误 → 修正

描述性标题改为观点句
饼图超 6 片合并或改条形
折线图比较项目改用条形图
Y 轴不从零还原零基线
多颜色装饰只保留区分色
3D 效果一律删除
一张图多信息拆成多张图
图例在角落直接标注在线旁
